Travel Planner API

Backend en NestJS para países y planes de viaje.

Endpoints principales

  • GET /countries
  • GET /countries/:code
  • DELETE /countries/:code
  • GET /travel-plans
  • GET /travel-plans/:id
  • POST /travel-plans

Notas

  • Ejemplo de país: /countries/COL
  • El endpoint DELETE /countries/:code está protegido: requiere el header x-delete-token con el valor configurado en la variable COUNTRIES_DELETE_TOKEN del archivo .env.
  • Las rutas /countries y /travel-plans están registradas por un middleware de logging que registra método, ruta, código de estado y tiempo de respuesta en la consola del servidor.
